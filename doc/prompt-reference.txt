1. INTRODUCTION                                          *prompt-reference-intro*

prompt-reference.nvim lets you stage code references - each paired with its own
prompt - into a "review", then hand the whole review to a sink: a register (the
system clipboard by default), or straight into a tmux pane running Claude Code.
Reviews are formatted as markdown or XML for pasting into an LLM.

Requires Neovim 0.10+ (uses |getregion()|).

==============================================================================
2. USAGE                                                 *prompt-reference-usage*

1. Select code in visual mode and add it (see |prompt-reference-commands| or
   the opt-in keymaps). A prompt window opens; type the instruction for that
   selection and press <CR>.
2. Repeat across files/regions. A review panel appears bottom-right showing the
   staged items.
3. Open the review and press <CR> to send the bundle to the configured sink
   (clipboard by default) and clear it.

Review window keys:~
  <CR>                send the whole review to the sink & clear
  dd                  delete the item under the cursor
  r                   re-enter the prompt for the item (shows code as context)
  ?                   show help
  <Tab><Tab>, <Esc>   close the review

==============================================================================
3. COMMANDS                                           *prompt-reference-commands*

                                                            *:PromptReferenceAdd*
:PromptReferenceAdd     Add the current visual selection to the review; opens a
                        prompt window for the per-item instruction.

                                                         *:PromptReferenceReview*
:PromptReferenceReview  Open the review window.

                                                           *:PromptReferenceCopy*
:PromptReferenceCopy    Copy the review to the register and clear it.

==============================================================================
4. CONFIGURATION                                        *prompt-reference-config*

Call setup() to change options and (optionally) enable default keymaps: >

    require("prompt-reference").setup({
      sink = "clipboard",        -- "clipboard" or "tmux"
      register = "+",
      use_git_root = true,
      include_code = true,
      output_style = "markdown", -- or "xml"
      tmux = {                   -- only used when sink = "tmux"
        command = "claude",      -- foreground command of the pane to auto-detect
        submit = true,           -- press Enter after pasting
        select = false,          -- switch focus to that pane after sending
      },
      keymaps = {
        add = "<CR>",          -- (visual) add selection to the review
        review = "<Tab><Tab>", -- (normal) open the review
        copy = false,          -- (normal) copy without opening the review
      },
    })
<
Options:~
  sink          string   "clipboard" (copy to register) or "tmux" (paste into
                         the pane running Claude Code). Default "clipboard".
  register      string   Register to copy into. Default "+".
  use_git_root  boolean  Paths relative to the git root, else cwd. Default true.
  include_code  boolean  Include the selected code, not just the ref. Default true.
  output_style  string   "markdown" or "xml". Default "markdown".
  tmux.command  string   Foreground command to match when auto-detecting the
                         target pane. Default "claude".
  tmux.submit   boolean  Press Enter after pasting so the review is sent.
                         Default true.
  tmux.select   boolean  Switch focus to the target pane after sending.
                         Default false.
  keymaps       tbl|bool `true` uses the defaults above; a table merges over
                         them; `false` (default) binds nothing.

tmux sink:~
With sink = "tmux", finishing a review pastes the whole bundle into the tmux
pane running Claude Code and presses Enter, instead of copying to a register.
Run Neovim and claude in the same tmux session; the target pane is found by
matching |tmux.command| against each pane's foreground command. The paste uses
bracketed paste so multi-line reviews arrive as one block. If Neovim isn't in
tmux, or no matching pane is found, the review falls back to the clipboard with
a warning. This is a one-way hand-off; it does not read Claude's response back.

==============================================================================
5. API                                                     *prompt-reference-api*

require("prompt-reference").add_selection()   Stage the visual selection.
require("prompt-reference").review()          Open the review window.
require("prompt-reference").copy_all()        Copy the review and clear it.
